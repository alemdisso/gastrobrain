# Issue 9: Standardize Navigation Elements

## Prompt for Agent

Create a GitHub issue to standardize navigation elements across the app.

**Context:**
- Consistent navigation styling improves usability and polish
- Includes bottom navigation bar, app bars, drawers (if applicable)
- Should follow design tokens and visual identity
- Creates cohesive navigation experience

**Issue Requirements:**
- Title: "Standardize Navigation Element Styles"
- Labels: `enhancement`, `UI`, `P3-Low`
- Milestone: `0.1.7 - Visual Foundations & Polish`
- Depends on: Issue #3 (ThemeData Implementation)

**Issue Description Should Include:**

1. **Objective**: Apply consistent styling to all navigation elements for cohesive user experience

2. **Navigation Elements to Standardize**:
   - Bottom navigation bar (Receitas, Planejamento, Ingredientes, Tools)
   - App bar / top navigation
   - Drawer (if applicable)
   - Back buttons
   - Tab bars (if applicable)

3. **Tasks**:
   - Review current navigation styling
   - Apply design tokens to navigation elements
   - Standardize icon styling and sizing
   - Configure active/inactive states
   - Ensure consistent elevation/shadows
   - Polish animations/transitions if applicable
   - Verify responsive behavior
   - Update navigation theme configuration

4. **Styling Requirements**:
   - Follow design tokens (colors, spacing, typography)
   - Clear active/inactive state indicators
   - Consistent with Gastrobrain's personality
   - Appropriate icon sizing and spacing
   - Smooth transitions

5. **Success Criteria**:
   - All navigation elements follow design tokens
   - Clear visual feedback for active states
   - Consistent styling across all navigation types
   - Smooth user experience when navigating
   - No hardcoded navigation styles
   - All tests passing

6. **Deliverable**:
   - Updated navigation theme configuration
   - All navigation elements updated
   - Documentation of navigation patterns
   - All tests passing
